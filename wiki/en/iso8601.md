# ISO 8601 rules

These rules validate [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) **date**, **datetime** and **duration** strings. Three of them build directly on the ISO 8601 helper functions shipped by [`oihana/php-standards`](https://packagist.org/packages/oihana/php-standards) — `org\iso\helpers\isIso8601DateTime()` and `org\iso\helpers\isIso8601Duration()` — so calendar validity (rejecting `2026-02-30`, honoring leap years) is enforced for you.

| Rule | Date | Time | Duration | Example value | Rule name |
|---|:---:|:---:|:---:|---|---|
| `ISO8601DateRule` | yes | optional | no | `2024-01-15` | *(no preset — name it yourself)* |
| `ISO8601DateTimeRule` | yes | required | no | `2026-05-14T08:15:30Z` | `Rules::ISO8601_DATE_TIME` (`iso8601_date_time`) |
| `ISO8601DurationRule` | no | no | yes | `P30D` | `Rules::ISO8601_DURATION` (`iso8601_duration`) |
| `ISO8601DateTimeOrDurationRule` | no | required | yes | `2027-01-01T00:00:00Z` / `P30D` | `Rules::ISO8601_DATE_TIME_OR_DURATION` (`iso8601_date_time_or_duration`) |

> **Empty values** — `ISO8601DateTimeRule`, `ISO8601DurationRule` and `ISO8601DateTimeOrDurationRule` treat `null` and `''` as **valid** (shape-only rules); pair them with `required` when the field is mandatory. `ISO8601DateRule` is the exception: it treats `null` and `''` as **invalid**.

## `ISO8601DateRule`

Validates an ISO 8601 **date** expression using a regular expression. It accepts date-only values, but also year-only, week dates, ordinal dates, and full datetime values with timezone offsets — the time component is optional.

- Valid: `2024-01-15`, `2024`, `2024-W05-1`, `2024-01-15T10:30:00Z`, `2024-01-15T10:30:00+02:00`
- Invalid: `not-a-date`, `2024-13-01` (month out of range), `''`, `null`

This rule has **no preset constant** in `Rules` and is not auto-registered: pick your own name when adding it to the `Factory`. You can also pass a custom pattern and message to the constructor.

```php
use oihana\validations\rules\ISO8601DateRule;

$rule = new ISO8601DateRule();

$rule->check( '2024-01-15' );  // true
$rule->check( '2024-W05-1' );  // true
$rule->check( '2024-13-01' );  // false (invalid month)
$rule->check( '' );            // false
```

Registered with the Somnambulist `Factory` under a name of your choice:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\ISO8601DateRule;

$factory = new Factory();
$factory->addRule( 'iso8601_date' , new ISO8601DateRule() );

$validation = $factory->validate(
    [ 'birthday' => '1990-05-01' ],
    [ 'birthday' => 'required|iso8601_date' ]
);

$validation->passes(); // true
```

A custom pattern and message can be supplied:

```php
$rule = new ISO8601DateRule( '/^\d{4}$/' , 'Year only, please.' );
$rule->check( '2024' );       // true
$rule->check( '2024-01-15' ); // false
```

## `ISO8601DateTimeRule`

Validates an ISO 8601 **datetime** expression. Delegates to `org\iso\helpers\isIso8601DateTime()`, so calendar validity is checked. A date *and* a time component are both required.

- Valid: `2026-05-14T08:15:30Z`, `2026-05-14T08:15:30+02:00`, `2026-05-14T08:15:30.123Z`, `2026-05-14T08:15:30-05:30`
- Invalid: `2026-05-14` (date only — use `ISO8601DateRule`), `2026-02-30T00:00:00Z` (invalid calendar date), `2026-05-14 08:15:30` (space separator, in strict mode)

The constructor accepts `bool $strict = true`. In strict mode (the default) the `T` separator is mandatory; with `$strict = false` a space separator (`2026-05-14 08:15:30`) is also accepted.

```php
use oihana\validations\rules\ISO8601DateTimeRule;

$rule = new ISO8601DateTimeRule();

$rule->check( '2026-05-14T08:15:30Z' );      // true
$rule->check( '2026-05-14T08:15:30.123Z' );  // true
$rule->check( '2026-05-14' );                // false (date only)
$rule->check( '2026-02-30T00:00:00Z' );      // false (invalid calendar date)

// Loose mode: accept a space instead of the T separator
$loose = new ISO8601DateTimeRule( strict: false );
$loose->check( '2026-05-14 08:15:30' );      // true
```

Registered with the `Factory` under its preset name:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DateTimeRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DATE_TIME , new ISO8601DateTimeRule() );

$validation = $factory->validate(
    [ 'published_at' => '2026-05-14T08:15:30Z' ],
    [ 'published_at' => 'required|iso8601_date_time' ]
);

$validation->passes(); // true
```

## `ISO8601DurationRule`

Validates an ISO 8601 **duration** string of the form `P[n]Y[n]M[n]W[n]DT[n]H[n]M[n]S`. Delegates to `org\iso\helpers\isIso8601Duration()`.

- Valid: `P1Y2M3D`, `PT4H30M`, `P1W`, `P30D`, `P0D`
- Invalid: `P` (no components), `PT` (`T` with no time components), `1Y2M` (missing `P`), `P1.5Y` (decimals, in strict mode)

The constructor accepts `bool $strict = true`. In strict mode (the default) validation is regex-based: at least one component is required and decimals are rejected. With `$strict = false` the rule delegates to PHP's `DateInterval` parser, which is more permissive.

```php
use oihana\validations\rules\ISO8601DurationRule;

$rule = new ISO8601DurationRule();

$rule->check( 'P1Y2M3D' ); // true
$rule->check( 'PT4H30M' ); // true
$rule->check( 'P30D' );    // true
$rule->check( 'P' );       // false (no components)
$rule->check( 'P1.5Y' );   // false (decimals rejected in strict mode)
```

Registered with the `Factory` under its preset name:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DurationRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DURATION , new ISO8601DurationRule() );

$validation = $factory->validate(
    [ 'ttl' => 'P30D' ],
    [ 'ttl' => 'required|iso8601_duration' ]
);

$validation->passes(); // true
```

## `ISO8601DateTimeOrDurationRule`

Validates a value that is **either** a valid ISO 8601 datetime **or** a valid ISO 8601 duration. Useful for fields that accept both an absolute deadline (`2027-01-01T00:00:00Z`) and a relative offset (`P30D`). It tries the duration branch first, then the datetime branch — both delegate to the `org\iso\helpers` functions.

Note that a pure calendar **date** (`2026-05-14`, without a time component) is **not** accepted; combine with `ISO8601DateRule` in a sibling rule chain if the field also accepts plain dates.

- Valid: `2027-01-01T00:00:00Z`, `2026-05-14T08:15:30+02:00`, `P30D`, `P1Y`, `PT1H`, `P1Y2M3D`
- Invalid: `2026-05-14` (date only), `not-a-date`, `P`, `1Y2M`, `2026-02-30T00:00:00Z`

The constructor accepts `bool $strict = true`, which applies to both branches (mandatory `T` separator on the datetime branch, regex validation on the duration branch).

```php
use oihana\validations\rules\ISO8601DateTimeOrDurationRule;

$rule = new ISO8601DateTimeOrDurationRule();

$rule->check( '2027-01-01T00:00:00Z' ); // true (datetime branch)
$rule->check( 'P30D' );                 // true (duration branch)
$rule->check( '2026-05-14' );           // false (date only, not accepted)
$rule->check( '1Y2M' );                 // false
```

Registered with the `Factory` under its preset name:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DateTimeOrDurationRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DATE_TIME_OR_DURATION , new ISO8601DateTimeOrDurationRule() );

$validation = $factory->validate(
    [ 'expires' => 'P30D' ],
    [ 'expires' => 'required|iso8601_date_time_or_duration' ]
);

$validation->passes(); // true
```
