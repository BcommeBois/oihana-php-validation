# i18n & postal rules

![Language](https://img.shields.io/badge/language-English-blue)

Two rules cover internationalized data: `I18nRule` validates a localized map of
translations keyed by language code, and `PostalCodeRule` validates a postal
code against the pattern of a given country. Both live in the namespace
`oihana\validations\rules` and extend the Somnambulist `Rule` base class, so they
can be used standalone via `check()` or registered with a `Factory` under a rule
name.

## `I18nRule`

`oihana\validations\rules\I18nRule` validates that a value is a localized map —
an array (or object) keyed by **language codes** — where:

- only the **allowed** language codes are present, and
- every value is either a `string` or `null`.

### Constructor

```php
use oihana\validations\rules\I18nRule;

// Restrict to French and English
$rule = new I18nRule( [ 'fr', 'en' ] );
```

The constructor accepts an array of codes, a single code string, or `null`. The
codes are lowercased internally, so `[ 'FR', 'EN' ]` and `[ 'fr', 'en' ]` are
equivalent. The same list can also be set after construction with the fluent
`languages()` method.

### What passes and what fails

Given a rule allowing `fr` and `en`:

| Value | Result | Why |
|---|---|---|
| `[ 'fr' => 'Bonjour', 'en' => 'Hello' ]` | passes | all keys allowed, values are strings |
| `[ 'fr' => null, 'en' => 'Hello' ]` | passes | `null` values are allowed |
| `(object) [ 'fr' => 'Bonjour', 'en' => 'Hello' ]` | passes | objects are cast to arrays |
| `[]` (empty array) | passes | no disallowed key is present |
| `[ 'fr' => 'Bonjour', 'it' => 'Ciao' ]` | fails | `it` is not in the allowed list |
| `[ 'fr' => 'Bonjour', 'en' => 123 ]` | fails | `123` is neither a string nor `null` |
| `'hello'` / `42` | fails | not an array or object |

Note that an empty map passes: the rule only rejects *disallowed* keys and
non-string/non-null values, so emptiness is considered valid. Combine it with
`required` if a non-empty value is expected.

### Registered with the `Factory`

Register the rule under a name and use it inside a rule string, exactly as in
`I18nRuleTest`:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\I18nRule;

$validator = new Factory();
$validator->addRule( 'i18n', new I18nRule( [ 'fr', 'en' ] ) );

// Valid: only allowed languages, string values
$payload    = [ 'description' => [ 'fr' => 'Bonjour', 'en' => 'Hello' ] ];
$validation = $validator->validate( $payload, [ 'description' => 'required|array|i18n' ] );

$validation->passes(); // true

// Invalid: 'it' is not an allowed language
$payload    = [ 'description' => [ 'fr' => 'Bonjour', 'it' => 'Ciao' ] ];
$validation = $validator->validate( $payload, [ 'description' => 'required|i18n' ] );

$validation->passes(); // false
```

The `array` token in `'required|array|i18n'` is an ordinary Somnambulist rule
that asserts the value is an array before `i18n` inspects its keys.

When validation fails the rule reports:
`:attribute contains invalid translations or unsupported languages.`

## `PostalCodeRule`

`oihana\validations\rules\PostalCodeRule` validates a postal code against the
regular-expression pattern of one or more countries. The patterns come from the
`oihana\enums\PostalCodePattern` enumeration, and countries are identified by
their **ISO 3166-1 alpha-2** code via `org\iso\ISO3166_1`.

### Country / pattern mechanism

For each candidate country, the rule looks up the matching regex with
`PostalCodePattern::getPattern( $country )` and tests the value with
`PostalCodePattern::isValid( $value, $country )`. A value passes as soon as it
matches **any** of the configured countries' patterns. A few examples of the
underlying patterns:

| Country | ISO code | Example that matches |
|---|---|---|
| France | `FR` | `75015` |
| United States | `US` | `90210`, `12345-6789` |
| Italy | `IT` | `00100` |
| Switzerland | `CH` | `1000` |
| Germany | `DE` | `10115` |

Countries with **no postal-code system** (for example `HK`, `MO`, `QA`, `TD`,
`UG`) have no pattern: `getPattern()` returns `null` and any value is therefore
rejected for them.

### Default behaviour

The constructor signature is:

```php
public function __construct(
    array|string|null $country = null,
    string            $default = ISO3166_1::FR,
)
```

- The `$default` country is `ISO3166_1::FR` (`'FR'`). If a `$default` is passed
  that `PostalCodePattern` does not know, it falls back to `ISO3166_1::FR`.
- The `$country` argument may be a single code, a list of codes, or `null`. When
  it is `null`, the rule uses `$default`, so a `PostalCodeRule` with no arguments
  validates **French** postal codes.
- Country codes are uppercased internally, so `'fr'` and `'FR'` are equivalent.
- The value being validated must be a `string`. A non-string value (an `int`,
  `null`, an array, …) always fails — `'75015'` passes for France, but the
  integer `75015` does not.

### Standalone example

```php
use oihana\validations\rules\PostalCodeRule;

// No argument → defaults to France
$rule = new PostalCodeRule();
$rule->check( '75015' ); // true
$rule->check( 'ABCDE' ); // false

// Explicit single country
$rule = new PostalCodeRule( 'US' );
$rule->check( '90210' );      // true
$rule->check( '12345-6789' ); // true

// Several countries — passes if any pattern matches
$rule = new PostalCodeRule()->country( [ 'FR', 'US' ] );
$rule->check( '75015' ); // true (FR)
$rule->check( '90210' ); // true (US)
```

### Registered with the `Factory`

Registered under a name, the rule can also receive its country as a rule-string
parameter:

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\PostalCodeRule;

$validator = new Factory();
$validator->addRule( 'postalCode', new PostalCodeRule() );

// Uses the rule's default country (France)
$validation = $validator->validate(
    [ 'postalCode' => '75015' ],
    [ 'postalCode' => 'required|postalCode' ]
);
$validation->passes(); // true

// Override the country in the rule string (ISO 3166-1 alpha-2)
$validation = $validator->validate(
    [ 'postalCode' => '00100' ],
    [ 'postalCode' => 'required|postalCode:IT' ]
);
$validation->passes(); // true
```

When validation fails the rule reports:
`:attribute is not a valid postal code with the value ':value'.`
