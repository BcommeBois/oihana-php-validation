# Règles i18n & postales

![Langue](https://img.shields.io/badge/langue-Français-blue)

Deux règles couvrent les données internationalisées : `I18nRule` valide une carte
localisée de traductions indexée par code de langue, et `PostalCodeRule` valide
un code postal selon le motif d'un pays donné. Les deux résident dans l'espace de
noms `oihana\validations\rules` et étendent la classe de base `Rule` de
Somnambulist : elles peuvent donc être utilisées seules via `check()` ou
enregistrées dans une `Factory` sous un nom de règle.

## `I18nRule`

`oihana\validations\rules\I18nRule` valide qu'une valeur est une carte localisée —
un tableau (ou un objet) indexé par **codes de langue** — où :

- seuls les codes de langue **autorisés** sont présents, et
- chaque valeur est soit une `string`, soit `null`.

### Constructeur

```php
use oihana\validations\rules\I18nRule;

// Restreindre au français et à l'anglais
$rule = new I18nRule( [ 'fr', 'en' ] );
```

Le constructeur accepte un tableau de codes, une chaîne de code unique ou `null`.
Les codes sont mis en minuscules en interne, si bien que `[ 'FR', 'EN' ]` et
`[ 'fr', 'en' ]` sont équivalents. La même liste peut aussi être définie après la
construction via la méthode fluide `languages()`.

### Ce qui passe et ce qui échoue

Pour une règle autorisant `fr` et `en` :

| Valeur | Résultat | Pourquoi |
|---|---|---|
| `[ 'fr' => 'Bonjour', 'en' => 'Hello' ]` | passe | toutes les clés autorisées, valeurs de type chaîne |
| `[ 'fr' => null, 'en' => 'Hello' ]` | passe | les valeurs `null` sont autorisées |
| `(object) [ 'fr' => 'Bonjour', 'en' => 'Hello' ]` | passe | les objets sont convertis en tableaux |
| `[]` (tableau vide) | passe | aucune clé interdite n'est présente |
| `[ 'fr' => 'Bonjour', 'it' => 'Ciao' ]` | échoue | `it` ne fait pas partie de la liste autorisée |
| `[ 'fr' => 'Bonjour', 'en' => 123 ]` | échoue | `123` n'est ni une chaîne ni `null` |
| `'hello'` / `42` | échoue | n'est ni un tableau ni un objet |

Notez qu'une carte vide passe : la règle ne rejette que les clés *interdites* et
les valeurs qui ne sont ni des chaînes ni `null`, la vacuité est donc considérée
comme valide. Combinez-la avec `required` si une valeur non vide est attendue.

### Enregistrée dans la `Factory`

Enregistrez la règle sous un nom et utilisez-la dans une chaîne de règles,
exactement comme dans `I18nRuleTest` :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\I18nRule;

$validator = new Factory();
$validator->addRule( 'i18n', new I18nRule( [ 'fr', 'en' ] ) );

// Valide : uniquement des langues autorisées, valeurs de type chaîne
$payload    = [ 'description' => [ 'fr' => 'Bonjour', 'en' => 'Hello' ] ];
$validation = $validator->validate( $payload, [ 'description' => 'required|array|i18n' ] );

$validation->passes(); // true

// Invalide : 'it' n'est pas une langue autorisée
$payload    = [ 'description' => [ 'fr' => 'Bonjour', 'it' => 'Ciao' ] ];
$validation = $validator->validate( $payload, [ 'description' => 'required|i18n' ] );

$validation->passes(); // false
```

Le jeton `array` dans `'required|array|i18n'` est une règle Somnambulist
ordinaire qui vérifie que la valeur est un tableau avant que `i18n` n'en inspecte
les clés.

En cas d'échec de la validation, la règle signale :
`:attribute contains invalid translations or unsupported languages.`

## `PostalCodeRule`

`oihana\validations\rules\PostalCodeRule` valide un code postal selon le motif
d'expression régulière d'un ou plusieurs pays. Les motifs proviennent de
l'énumération `oihana\enums\PostalCodePattern`, et les pays sont identifiés par
leur code **ISO 3166-1 alpha-2** via `org\iso\ISO3166_1`.

### Mécanisme pays / motif

Pour chaque pays candidat, la règle recherche l'expression régulière
correspondante avec `PostalCodePattern::getPattern( $country )` et teste la valeur
avec `PostalCodePattern::isValid( $value, $country )`. Une valeur passe dès
qu'elle correspond au motif de **l'un** des pays configurés. Quelques exemples des
motifs sous-jacents :

| Pays | Code ISO | Exemple correspondant |
|---|---|---|
| France | `FR` | `75015` |
| États-Unis | `US` | `90210`, `12345-6789` |
| Italie | `IT` | `00100` |
| Suisse | `CH` | `1000` |
| Allemagne | `DE` | `10115` |

Les pays **sans système de code postal** (par exemple `HK`, `MO`, `QA`, `TD`,
`UG`) n'ont aucun motif : `getPattern()` renvoie `null` et toute valeur est donc
rejetée pour eux.

### Comportement par défaut

La signature du constructeur est :

```php
public function __construct(
    array|string|null $country = null,
    string            $default = ISO3166_1::FR,
)
```

- Le pays `$default` est `ISO3166_1::FR` (`'FR'`). Si un `$default` inconnu de
  `PostalCodePattern` est passé, il revient à `ISO3166_1::FR`.
- L'argument `$country` peut être un code unique, une liste de codes ou `null`.
  Lorsqu'il vaut `null`, la règle utilise `$default` : un `PostalCodeRule` sans
  argument valide donc les codes postaux **français**.
- Les codes pays sont mis en majuscules en interne, si bien que `'fr'` et `'FR'`
  sont équivalents.
- La valeur validée doit être une `string`. Une valeur non chaîne (un `int`,
  `null`, un tableau, …) échoue toujours — `'75015'` passe pour la France, mais
  l'entier `75015` non.

### Exemple autonome

```php
use oihana\validations\rules\PostalCodeRule;

// Sans argument → par défaut la France
$rule = new PostalCodeRule();
$rule->check( '75015' ); // true
$rule->check( 'ABCDE' ); // false

// Pays unique explicite
$rule = new PostalCodeRule( 'US' );
$rule->check( '90210' );      // true
$rule->check( '12345-6789' ); // true

// Plusieurs pays — passe si un motif correspond
$rule = new PostalCodeRule()->country( [ 'FR', 'US' ] );
$rule->check( '75015' ); // true (FR)
$rule->check( '90210' ); // true (US)
```

### Enregistrée dans la `Factory`

Enregistrée sous un nom, la règle peut aussi recevoir son pays sous forme de
paramètre dans la chaîne de règles :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\PostalCodeRule;

$validator = new Factory();
$validator->addRule( 'postalCode', new PostalCodeRule() );

// Utilise le pays par défaut de la règle (la France)
$validation = $validator->validate(
    [ 'postalCode' => '75015' ],
    [ 'postalCode' => 'required|postalCode' ]
);
$validation->passes(); // true

// Surcharger le pays dans la chaîne de règles (ISO 3166-1 alpha-2)
$validation = $validator->validate(
    [ 'postalCode' => '00100' ],
    [ 'postalCode' => 'required|postalCode:IT' ]
);
$validation->passes(); // true
```

En cas d'échec de la validation, la règle signale :
`:attribute is not a valid postal code with the value ':value'.`
