# Règles ISO 8601

Ces règles valident les chaînes de **date**, **datetime** et **durée** au format [ISO 8601](https://fr.wikipedia.org/wiki/ISO_8601). Trois d'entre elles s'appuient directement sur les fonctions d'aide ISO 8601 fournies par [`oihana/php-standards`](https://packagist.org/packages/oihana/php-standards) — `org\iso\helpers\isIso8601DateTime()` et `org\iso\helpers\isIso8601Duration()` — de sorte que la validité calendaire (rejet de `2026-02-30`, prise en compte des années bissextiles) est garantie pour vous.

| Règle | Date | Heure | Durée | Exemple de valeur | Nom de règle |
|---|:---:|:---:|:---:|---|---|
| `ISO8601DateRule` | oui | optionnelle | non | `2024-01-15` | *(aucun préréglage — à nommer vous-même)* |
| `ISO8601DateTimeRule` | oui | requise | non | `2026-05-14T08:15:30Z` | `Rules::ISO8601_DATE_TIME` (`iso8601_date_time`) |
| `ISO8601DurationRule` | non | non | oui | `P30D` | `Rules::ISO8601_DURATION` (`iso8601_duration`) |
| `ISO8601DateTimeOrDurationRule` | non | requise | oui | `2027-01-01T00:00:00Z` / `P30D` | `Rules::ISO8601_DATE_TIME_OR_DURATION` (`iso8601_date_time_or_duration`) |

> **Valeurs vides** — `ISO8601DateTimeRule`, `ISO8601DurationRule` et `ISO8601DateTimeOrDurationRule` considèrent `null` et `''` comme **valides** (règles de forme uniquement) ; associez-les à `required` lorsque le champ est obligatoire. `ISO8601DateRule` fait exception : elle considère `null` et `''` comme **invalides**.

## `ISO8601DateRule`

Valide une expression de **date** ISO 8601 à l'aide d'une expression régulière. Elle accepte les valeurs date seule, mais aussi année seule, dates de semaine, dates ordinales et valeurs datetime complètes avec décalage horaire — la partie heure est optionnelle.

- Valides : `2024-01-15`, `2024`, `2024-W05-1`, `2024-01-15T10:30:00Z`, `2024-01-15T10:30:00+02:00`
- Invalides : `not-a-date`, `2024-13-01` (mois hors plage), `''`, `null`

Cette règle n'a **aucune constante préréglée** dans `Rules` et n'est pas enregistrée automatiquement : choisissez votre propre nom au moment de l'ajouter à la `Factory`. Vous pouvez également passer un motif et un message personnalisés au constructeur.

```php
use oihana\validations\rules\ISO8601DateRule;

$rule = new ISO8601DateRule();

$rule->check( '2024-01-15' );  // true
$rule->check( '2024-W05-1' );  // true
$rule->check( '2024-13-01' );  // false (mois invalide)
$rule->check( '' );            // false
```

Enregistrée auprès de la `Factory` de Somnambulist sous un nom de votre choix :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\rules\ISO8601DateRule;

$factory = new Factory();
$factory->addRule( 'iso8601_date' , new ISO8601DateRule() );

$validation = $factory->validate(
    [ 'birthday' => '1990-05-01' ],
    [ 'birthday' => 'required|iso8601_date' ]
);

$validation->passes(); // true
```

Un motif et un message personnalisés peuvent être fournis :

```php
$rule = new ISO8601DateRule( '/^\d{4}$/' , 'Year only, please.' );
$rule->check( '2024' );       // true
$rule->check( '2024-01-15' ); // false
```

## `ISO8601DateTimeRule`

Valide une expression **datetime** ISO 8601. Délègue à `org\iso\helpers\isIso8601DateTime()`, de sorte que la validité calendaire est vérifiée. Une partie date *et* une partie heure sont toutes deux requises.

- Valides : `2026-05-14T08:15:30Z`, `2026-05-14T08:15:30+02:00`, `2026-05-14T08:15:30.123Z`, `2026-05-14T08:15:30-05:30`
- Invalides : `2026-05-14` (date seule — utilisez `ISO8601DateRule`), `2026-02-30T00:00:00Z` (date calendaire invalide), `2026-05-14 08:15:30` (séparateur espace, en mode strict)

Le constructeur accepte `bool $strict = true`. En mode strict (par défaut), le séparateur `T` est obligatoire ; avec `$strict = false`, un séparateur espace (`2026-05-14 08:15:30`) est également accepté.

```php
use oihana\validations\rules\ISO8601DateTimeRule;

$rule = new ISO8601DateTimeRule();

$rule->check( '2026-05-14T08:15:30Z' );      // true
$rule->check( '2026-05-14T08:15:30.123Z' );  // true
$rule->check( '2026-05-14' );                // false (date seule)
$rule->check( '2026-02-30T00:00:00Z' );      // false (date calendaire invalide)

// Mode souple : accepte un espace à la place du séparateur T
$loose = new ISO8601DateTimeRule( strict: false );
$loose->check( '2026-05-14 08:15:30' );      // true
```

Enregistrée auprès de la `Factory` sous son nom préréglé :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DateTimeRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DATE_TIME , new ISO8601DateTimeRule() );

$validation = $factory->validate(
    [ 'published_at' => '2026-05-14T08:15:30Z' ],
    [ 'published_at' => 'required|iso8601_date_time' ]
);

$validation->passes(); // true
```

## `ISO8601DurationRule`

Valide une chaîne de **durée** ISO 8601 de la forme `P[n]Y[n]M[n]W[n]DT[n]H[n]M[n]S`. Délègue à `org\iso\helpers\isIso8601Duration()`.

- Valides : `P1Y2M3D`, `PT4H30M`, `P1W`, `P30D`, `P0D`
- Invalides : `P` (aucun composant), `PT` (`T` sans composant horaire), `1Y2M` (`P` manquant), `P1.5Y` (décimales, en mode strict)

Le constructeur accepte `bool $strict = true`. En mode strict (par défaut), la validation est basée sur une expression régulière : au moins un composant est requis et les décimales sont rejetées. Avec `$strict = false`, la règle délègue à l'analyseur `DateInterval` de PHP, plus permissif.

```php
use oihana\validations\rules\ISO8601DurationRule;

$rule = new ISO8601DurationRule();

$rule->check( 'P1Y2M3D' ); // true
$rule->check( 'PT4H30M' ); // true
$rule->check( 'P30D' );    // true
$rule->check( 'P' );       // false (aucun composant)
$rule->check( 'P1.5Y' );   // false (décimales rejetées en mode strict)
```

Enregistrée auprès de la `Factory` sous son nom préréglé :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DurationRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DURATION , new ISO8601DurationRule() );

$validation = $factory->validate(
    [ 'ttl' => 'P30D' ],
    [ 'ttl' => 'required|iso8601_duration' ]
);

$validation->passes(); // true
```

## `ISO8601DateTimeOrDurationRule`

Valide une valeur qui est **soit** une datetime ISO 8601 valide, **soit** une durée ISO 8601 valide. Utile pour les champs qui acceptent à la fois une échéance absolue (`2027-01-01T00:00:00Z`) et un décalage relatif (`P30D`). La règle essaie d'abord la branche durée, puis la branche datetime — toutes deux déléguant aux fonctions `org\iso\helpers`.

Notez qu'une **date** calendaire pure (`2026-05-14`, sans partie heure) **n'est pas** acceptée ; combinez avec `ISO8601DateRule` dans une chaîne de règles sœur si le champ accepte aussi les dates simples.

- Valides : `2027-01-01T00:00:00Z`, `2026-05-14T08:15:30+02:00`, `P30D`, `P1Y`, `PT1H`, `P1Y2M3D`
- Invalides : `2026-05-14` (date seule), `not-a-date`, `P`, `1Y2M`, `2026-02-30T00:00:00Z`

Le constructeur accepte `bool $strict = true`, qui s'applique aux deux branches (séparateur `T` obligatoire sur la branche datetime, validation par expression régulière sur la branche durée).

```php
use oihana\validations\rules\ISO8601DateTimeOrDurationRule;

$rule = new ISO8601DateTimeOrDurationRule();

$rule->check( '2027-01-01T00:00:00Z' ); // true (branche datetime)
$rule->check( 'P30D' );                 // true (branche durée)
$rule->check( '2026-05-14' );           // false (date seule, non acceptée)
$rule->check( '1Y2M' );                 // false
```

Enregistrée auprès de la `Factory` sous son nom préréglé :

```php
use Somnambulist\Components\Validation\Factory;
use oihana\validations\enums\Rules;
use oihana\validations\rules\ISO8601DateTimeOrDurationRule;

$factory = new Factory();
$factory->addRule( Rules::ISO8601_DATE_TIME_OR_DURATION , new ISO8601DateTimeOrDurationRule() );

$validation = $factory->validate(
    [ 'expires' => 'P30D' ],
    [ 'expires' => 'required|iso8601_date_time_or_duration' ]
);

$validation->passes(); // true
```
